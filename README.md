# Album Tracker (基于 AI 协同开发的个性化专辑管理系统)

这是一个基于本人对专辑整理的需求，与Claude协同开发而成的个性化音乐专辑管理与数据分析系统。

本人出发点是为了解决海量专辑收藏场景下（如个人 2000+ 专辑储备），市面现有工具在流派精细化分类上的痛点。

## 核心功能与数据架构

1. **多源数据融合与对齐 (Data Convergence):**
   - 整合多个开源音乐数据库，全自动提取并补全专辑的【封面、名称、歌手、流派、曲目、年代】等多维核心元数据。
   - 经过实际数据测试，外部数据自动补全的**覆盖率可达 80%**。

2. **创新的“流派-子流派”双标签分类系统:**
   - 针对多元化的音乐风格，设计了“主流派 + 子流派”的双层标签治理逻辑，支持高自由度的矩阵式分类管理（本系统的核心硬需求）。
   - 同时支持按年代、歌手或名称等多维度建立动态索引与看板。

3. **数据自动化导入与匹配 (ETL Workflow):**
   - 支持一键导入标准 CSV 文件，系统后端会自动与开源数据库进行字段映射与记录匹配，实现资产的自动化上架。

4. **完善的资产评价与互动体系:**
   - 内置 10 星精细化评分机制，并集成短评功能。
   - 预留/实现了链接各主流乐评网站的跳转与数据参照功能。

5. **现代化的数据存储演进:**
   - 项目初期采用数据内嵌文件的本地化存储方案。随着架构迭代，目前正逐步向云端存储与高效管理演进，以确保数据的可靠性与多端同步能力。

目前系统已更新至 **V7 版本**，成功将前期的内嵌数据存储重构修改为**云数据库处理**，已完全满足个人的基础日常使用。但也发现了当前行业开源数据的一个客观痛点：
目前开源数据库整合的 18 类主流派 + 6-10 类子流派，在面对现代极其多元和融合的音乐风格（如各种微流派、跨界流派）时仍显不足。
正在进一步收集、清洗更精细的音乐风格数据集，用以优化后端的标签推荐算法与分类精度。
